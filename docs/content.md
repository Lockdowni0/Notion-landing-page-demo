## 14. Responsive Copy Rules

- Do not replace canonical headlines solely to make layout easier.
- Use defined short variants only inside utility UI.
- Preserve the exact original quotation on all viewports.
- On narrow screens, change line breaks rather than wording.
- Do not truncate project names silently; use wrapping first and ellipsis only inside compact controls.
- Dynamic copy must fall back gracefully when a project name is empty or unusually long.

## 15. Dynamic Copy Safety

- Treat visitor-provided project and task names as plain text, never HTML.
- Limit project name display to 80 characters in long-form headings and 32 characters in compact controls.
- Do not interpolate personal information into marketing copy.
- Do not infer personality, ability, or intent from Demo choices.
- Adapt only to explicit structural choices: name, view, fields, statuses, and tasks.
